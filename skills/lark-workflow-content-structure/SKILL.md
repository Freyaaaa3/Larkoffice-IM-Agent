# 内容结构化辅助

## 适用场景

- 需要将散乱的讨论内容整理为结构化文档
- 需要将长文本拆分为适合幻灯片展示的短内容
- 需要为不同类型演示选择合适的内容模板

## 核心原则

### 文档 vs 幻灯片的内容差异

| 维度 | 文档 | 幻灯片 |
|------|------|--------|
| 信息密度 | 高，可长段落 | 低，每页1个核心观点 |
| 列表项 | 可5-10项 | 不超过3-5项 |
| 详细程度 | 完整描述 | 关键词+要点 |
| 逻辑关系 | 详尽论证 | 视觉化呈现 |

### 内容提取步骤

1. **识别核心主题**：从讨论中提炼一个中心论点
2. **提取关键论据**：3-5个支撑论点
3. **补充数据/案例**：每个论据配1-2个数据点
4. **组织层次**：论点→子论点→证据

### 幻灯片适配规则

- 封面页：标题 + 副标题 + 日期/作者
- 目录页：3-5个章节标题
- 内容页：标题 + 3-5个要点（每点1行）
- 数据页：1个大数字 + 简短说明
- 对比页：2-3列并排内容
- 结尾页：总结 + 行动号召

## 常见演示类型模板

### 项目汇报

```
1. 封面：项目名称 + 汇报人
2. 项目概述：背景 + 目标
3. 进展回顾：关键里程碑（时间线）
4. 核心成果：3-5个成果数据
5. 挑战与应对：问题 + 解决方案
6. 下一步计划：3-5个行动项
7. 结尾：感谢 + Q&A
```

### 产品发布

```
1. 封面：产品名称 + Slogan
2. 痛点分析：3个用户痛点
3. 产品方案：核心功能展示
4. 技术架构：系统/流程图
5. 数据验证：关键指标对比
6. 路线图：后续规划
7. 结尾：呼吁行动
```

### 季度总结

```
1. 封面：季度 + 团队名称
2. 季度概览：3个核心数据
3. 重点项目：逐项目展示
4. 团队成长：能力提升
5. 问题复盘：经验教训
6. 下季度计划：3个重点方向
7. 结尾：感谢
```
